BLOTTER DEPLOY — www.tradeblotter.app
======================================
Updated 2026-07-12

WHAT THIS FOLDER IS
Your entire GitHub Pages repo root. The site is two pages:
  index.html    -> landing page (SEO + live demo embed)
  app.html      -> the Blotter app itself
  support.js    -> runtime the landing page loads (index.html needs it;
                   app.html is self-contained and does NOT)

WHAT'S NEW IN THIS BUILD
- Welcome screen simplified to ONE setup path: "Step 1 of 2 · Create
  your Flex Query" -> "Step 2 of 2 · Turn on auto-refresh". The old
  "Quickest" Transaction History card is gone — those CSVs still import
  fine (the parser is unchanged); a footnote under the cards says so.
- Activity log in Data & sync: every meaningful sync event — new
  trades pulled by auto-refresh (with the days affected), CSV imports,
  connect/disconnect, and refresh errors — last 30 events, stored on
  this device only (tz_sync_log). Header shows LAST CHECKED so users
  can confirm the automation is alive without reading the log.
- IBKR auto-refresh (Flex Web Service) — LIVE. The official relay is
  baked in (blotter-flex-relay.harry-m87.workers.dev), so users configure
  NOTHING infrastructure-side. Their whole setup is: create the Flex
  Query, enable Flex Web Service, paste Query ID + token into Blotter.
  Verify does a real IBKR round-trip (send -> poll -> parse -> reconcile);
  auto-refresh runs on open, on tab-return, and every 5 min while open.
  IBKR errors map to plain-language fixes (expired token, wrong query
  id, still-generating, etc).
  The RELAY section in Data & sync is hidden for normal users. It only
  appears when a self-hoster opens the app with #relay in the URL (e.g.
  app.html#relay) or already has an override saved — they can route
  through their own worker; clearing the field returns to the official
  relay. If you later serve the relay from a custom domain (e.g.
  relay.tradeblotter.app), ask Claude to update the baked-in default and
  re-bundle.
  A migration on load removes the sample dataset any earlier build saved
  under "IBKR · auto", so affected testers drop back to the welcome
  screen instead of seeing demo data as their own. The token is stored
  unencrypted — add passphrase encryption before shipping real
  credentials.
- Live backup file (File System Access) + one-time download fallback.
- Demo data is now a full year (Jul 2025 – Jun 2026, ~860 trades).
- Landing page, FAQ, and structured data updated for auto-refresh.

STEPS
0. Live IBKR sync works out of the box — the official relay
   (blotter-flex-relay.harry-m87.workers.dev, running relay/worker.js)
   is baked into app.html. Keep the relay/ folder in the repo so the
   code is public — link it from the FAQ. Free tier covers ~500 heavy
   daily users; upgrade at Cloudflare if invocations approach 100k/day.
1. In your repo, KEEP the CNAME file if one exists (it holds your custom
   domain). Do not delete it.
2. Upload everything in this folder to the repo root, overwriting the old
   files:
     index.html, app.html, support.js, og.png, robots.txt, sitemap.xml
3. Wait ~1 minute for Pages to rebuild, then hard-refresh
   (Cmd/Ctrl+Shift+R) on:
     https://www.tradeblotter.app/         -> landing page
     https://www.tradeblotter.app/app.html -> the app
4. Re-scan https://www.opengraph.xyz for both URLs.

NOTES
- Existing testers' data is safe: browser storage is per-domain, not
  per-page, so journals survive this update.
- The landing's "live demo" panel embeds app.html#demo — the app opens
  straight into sample data via that link. Use
  https://www.tradeblotter.app/app.html#demo in Reddit/X posts.
- Auto-refresh + live backup use browser APIs (File System Access) that
  need a real https:// origin — they work on the deployed site and in
  Chrome/Edge; other browsers get the download fallback.
- Google Search Console: resubmit sitemap.xml once deployed.
