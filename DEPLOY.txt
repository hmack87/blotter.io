BLOTTER DEPLOY — www.tradeblotter.app
======================================
Updated 2026-07-12

WHAT THIS FOLDER IS
Your entire GitHub Pages repo root. The site is two pages:
  index.html    -> landing page (SEO + live demo embed)
  app.html      -> the Blotter app itself
  support.js    -> runtime the landing page loads (index.html needs it;
                   app.html is self-contained and does NOT)

WHAT'S NEW IN THIS BUILD
- IBKR auto-refresh (Flex Web Service). Connect once from the welcome
  screen's "Automated" card or the header/footer "Data & sync". New
  trades flow in on open and every ~5 min while Blotter stays open
  (once the relay is live — see above). Setup validates the Query ID +
  token format up front.
- IBKR auto-refresh (Flex Web Service) — RELAY-READY BUILD. The connect
  flow validates and saves the Query ID + token on-device. Whether it
  fetches for real depends on one thing: the relay URL.
    * No relay configured -> honest "relay pending" state everywhere
      (chip, Data & sync, connect card). Never fakes data.
    * Relay configured -> real fetches: verify does a live IBKR round-trip
      (send -> poll -> parse -> reconcile), then auto-refresh runs on
      open, on tab-return, and every 5 min while open. IBKR errors are
      mapped to plain-language fixes (expired token, wrong query id, etc).
  TO GO LIVE: deploy relay/worker.js (free Cloudflare Worker — steps in
  relay/README.md), then in the app open Data & sync -> RELAY -> paste
  the worker URL -> Save. (Welcome screen: "Data & sync ->" link on the
  Auto-refresh card. Dashboard: footer link or header IBKR chip.)
  That's it — no rebuild needed. To bake the URL
  in for ALL users instead, ask Claude to set it as the default and
  re-bundle.
  A migration on load removes the sample dataset any earlier build saved
  under "IBKR · auto", so affected testers drop back to the welcome
  screen instead of seeing demo data as their own. The token is stored
  unencrypted — add passphrase encryption before shipping real
  credentials.
- Live backup file (File System Access) + one-time download fallback.
- Demo data is now a full year (Jul 2025 – Jun 2026, ~860 trades).
- Landing page, FAQ, and structured data updated for auto-refresh.

STEPS
0. To turn on live IBKR sync: deploy the relay first — see relay/README.md
   (free Cloudflare Worker, ~3 minutes, no tools needed). Then paste its
   URL into the app under Data & sync -> RELAY. The relay/ folder is
   included in this repo so the code is public — link it from the FAQ.
1. In your repo, KEEP the CNAME file if one exists (it holds your custom
   domain). Do not delete it.
2. Upload everything in this folder to the repo root, overwriting the old
   files:
     index.html, app.html, support.js, og.png, robots.txt, sitemap.xml
3. Wait ~1 minute for Pages to rebuild, then hard-refresh
   (Cmd/Ctrl+Shift+R) on:
     https://www.tradeblotter.app/         -> landing page
     https://www.tradeblotter.app/app.html -> the app
4. Re-scan https://www.opengraph.xyz for both URLs.

NOTES
- Existing testers' data is safe: browser storage is per-domain, not
  per-page, so journals survive this update.
- The landing's "live demo" panel embeds app.html#demo — the app opens
  straight into sample data via that link. Use
  https://www.tradeblotter.app/app.html#demo in Reddit/X posts.
- Auto-refresh + live backup use browser APIs (File System Access) that
  need a real https:// origin — they work on the deployed site and in
  Chrome/Edge; other browsers get the download fallback.
- Google Search Console: resubmit sitemap.xml once deployed.
